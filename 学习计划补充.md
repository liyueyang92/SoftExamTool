## 补充章节：学习计划模块详细设计

### 1. 核心数据结构

学习计划模块围绕以下核心实体组织数据。

#### 1.1 考试配置 (exam_config)
- **字段**：考试名称、考试日期、大纲版本、总分目标、每日可用学习时长范围。
- **说明**：全局唯一记录，用户初次使用时设定，后续可修改。

#### 1.2 章节/知识域 (knowledge_domain)
- **字段**：章节标识、名称、父级章节、建议学习时长（分钟）、权重（百分比）、是否必修。
- **来源**：内置软考系统架构设计师官方大纲，按三级结构拆解（如“系统设计 → 架构设计方法 → 面向服务架构”）。用户导入PDF教材后，AI可自动将教材章节映射至此结构。

#### 1.3 学习任务 (learning_task)
- **字段**：任务ID、关联章节、任务类型（阅读教材/观看视频/练习/复习/论文写作）、计划日期、预计耗时、实际耗时、完成状态（未开始/进行中/已完成/跳过）、优先级、关联题目数量。
- **子表**：练习任务可关联具体题目ID列表，复习任务可关联错题集合。

#### 1.4 学习记录 (learning_log)
- **字段**：日期、时段（上午/下午/晚上）、任务ID、专注时长、番茄钟周期数、中断次数、自评效率（1-5星）。
- **用途**：为动态调整和统计分析提供基础数据。

#### 1.5 计划模板 (plan_template)
- **字段**：模板名称、适用阶段（基础/强化/冲刺）、章节分配规则（JSON）、每日任务样例。
- **用途**：用户可保存和分享自定义计划模板，社区可提供专家备考模板（不含具体题目内容，仅框架）。

---

### 2. 计划自动生成算法

当用户设定考试日期后，系统按以下流程自动生成初始学习计划：

#### 2.1 阶段划分
根据剩余天数划分三个阶段（可配置默认比例）：
- **基础阶段**（剩余天数的 60%）：逐章学习教材，配合基础练习。
- **强化阶段**（剩余天数的 30%）：专项突破（案例、论文），高频错题重做。
- **冲刺阶段**（剩余天数的 10%）：全真模考，查漏补缺。

各阶段结束前预留 2~3 天缓冲，用于补漏和调整。

#### 2.2 任务生成逻辑
- **基础阶段**：
    - 按大纲顺序遍历所有知识域，根据 `建议学习时长` 和 `权重` 分配每日任务。
    - 每日任务组合遵循“先理论后练习”：每个章节依次生成“阅读教材（PDF指定章节）”任务、“基础练习（随机抽取对应知识域单选题 15 道）”任务、“错题回顾”任务（第二日起）。
    - 使用用户配置的每日可用时长约束总耗时，超出的任务自动顺延至次日。
- **强化阶段**：
    - 根据基础阶段做题记录，统计各知识域正确率，优先为正确率最低的 30% 章节生成“强化练习（中等难度+多选题）”和“案例分析专项”任务。
    - 每周安排 1~2 次“论文写作”任务，主题按历年高频考点轮换。
- **冲刺阶段**：
    - 隔日安排一次“全真模拟”（综合知识+案例分析+论文，严格计时）。
    - 模拟次日为“复盘日”，任务为查看AI评分报告、重做错题、复习薄弱知识点。

#### 2.3 任务时间估算
- 阅读教材：按页数估算（默认每分钟阅读2页），或根据PDF解析出的章节字数估算。
- 练习题：客观题每题 2 分钟，案例分析每题 15 分钟，论文每篇 120 分钟。
- 复习/回顾：预估为原任务耗时的 50%。

### 3. 动态调整机制

计划并非一成不变，系统持续收集学习数据并自动调整后续任务，但仍允许用户手动锁定部分计划。

#### 3.1 基于正确率的调整
- 每次练习后更新知识域掌握度模型（指数加权移动平均）。掌握度低于 40% 的章节自动标记为“薄弱”。
- 未来 3 天内自动增加该章节的“强化练习”任务，并减少已掌握章节的低价值重复练习。
- 若用户连续两次同一章节练习正确率 > 85%，系统建议将该章节设为“已掌握”，从后续计划中移除常规练习，仅保留考前复习。

#### 3.2 基于学习速度的调整
- 连续 3 日实际学习时长低于计划时长 30%，系统提示“进度滞后”，并自动压缩低权重章节任务量（如减少练习数量），或将部分任务切换为“快速复习”模式（仅查看摘要和错题）。
- 若用户超前完成计划且时间充裕，系统建议提前进入下一阶段，或生成“拓展阅读”（如行业白皮书、技术博客，由用户自行导入）。

#### 3.3 缓冲与赶工
- 自动生成的计划每周预留 1 天为“弹性日”，不安排固定任务，用于补漏或休息。
- 当用户标记某日“请假”后，系统将当日未完成任务按优先级分散到后续 3 天。
- 冲刺阶段开启“赶工模式”：用户可设置每日最大学习时长上限，系统自动裁剪次要任务，优先保证模考和错题重做。

#### 3.4 用户手动控制
- 支持拖拽任务到其他日期、修改任务预估耗时、添加自定义任务（如“观看某视频”）。
- “锁定”功能：可锁定未来 N 天的计划，锁定期内系统不会自动调整。
- 支持“重置计划”并重新生成，历史学习数据保留，仅重新分配任务。

---

### 4. 冲刺模式专项设计

距考试 30 天（用户可修改阈值）时自动激活冲刺模式，界面和任务逻辑发生改变：

- **视觉变化**：导航栏增加倒计时悬浮组件，主色切换为红色系以营造紧迫感。
- **任务优先级重新排序**：错题重做 > 案例分析 > 论文 > 全真模考 > 知识点记忆。
- **每日论文**：强制每天一篇论文写作（可选历年真题或AI生成模拟题），并于次日反馈AI评分，循环迭代。
- **高频考点速览**：根据本地题库中知识点出现频率和用户错误频率，动态生成“每日必背10大要点”卡片，在通知中推送。
- **睡眠提醒**：冲刺模式开启后，如果用户在凌晨 0:00~6:00 使用软件，弹出提醒建议休息。

---

### 5. 提醒与督促系统

#### 5.1 通知类型与触发规则
- **每日计划通知**：每天用户设定的学习开始时间（如 19:00）弹出，包含今日任务清单和预估总耗时。
- **进度滞后警告**：连续 2 天未完成 70% 以上任务时，在次日早晨提醒并提供“调整计划”入口。
- **连续打卡激励**：连续学习 3/7/21/30 天时，推送成就通知，可展示动画徽章。
- **考前冲刺提醒**：倒计时 7 天、3 天、1 天时，推送特殊鼓励语和重要注意事项。
- **番茄钟结束提醒**：一个番茄钟（默认 25 分钟）结束时，弹出短暂休息建议，并记录一个番茄周期。

#### 5.2 番茄钟与专注模式
- 用户可在学习任意任务时启动番茄钟，系统进入专注模式：
    - 屏蔽其他应用窗口（可选，通过 Windows API 保持前置）。
    - 禁用非必要通知，仅保留番茄钟计时悬浮窗。
    - 记录专注时长，中断次数（手动暂停或切换窗口超过 30 秒计为中断）。
- 统计数据用于生成“专注度报告”，帮助用户优化学习时段。

#### 5.3 成就系统
定义徽章清单，例如：
- **初次通关**：完成第一次模拟考试。
- **勤学苦练**：累计做题 1000 道。
- **笔耕不辍**：完成 10 篇论文写作。
- **知错能改**：错题重做正确率提升 20%。
- **知识渊博**：导入并阅读完 3 本教材。
- **完美主义**：某次模拟考试综合知识部分满分。
  徽章数据存储在本地，在主界面学习概览区域展示。

---

### 6. 进度统计与可视化

#### 6.1 学习概览仪表盘
- **核心指标卡片**：总学习天数、累计时长、总做题量、当前正确率均值、待复习错题数。
- **日历热力图**：每个日期按学习时长或完成任务比例显示颜色深浅，一目了然。
- **知识域雷达图**：按大纲一级领域展示当前掌握度（基于练习正确率加权计算）。
- **趋势折线图**：近期每日学习时长、正确率变化曲线。

#### 6.2 弱项分析报告
- 按三级知识域列出正确率最低的 5 项，提供“一键生成强化计划”按钮。
- 案例分析得分趋势，标注AI常扣分的维度（如“逻辑清晰度”）。
- 论文历史分数分布，展示常用案例素材库是否匮乏（AI可建议补充素材）。

### 7. 与 AI 模块的协同

- **智能任务推荐**：在每日计划底部，AI 根据近期学习数据推荐一个可选任务，例如“发现你对微服务架构理解薄弱，建议阅读教材第8章并完成以下5道额外题”。
- **学习路径咨询**：在 AI 对话页面，用户可提问“我下个月考试，现在分布式系统章节正确率只有 50%，该如何调整计划？”，AI 基于实际数据给出个性化建议。
- **论文素材关联**：当学习计划推进到某知识域时，AI 从用户导入的项目文档或过往论文中提取相关案例摘要，作为写作参考自动出现在论文写作任务旁。
- **计划模板生成**：用户可用自然语言描述需求（如“我只有晚上2小时，想重点突击案例分析和论文”），AI 生成自定义计划模板供用户微调后使用。

#### 11.1 三阶段计划生成

```typescript
function generatePhasedPlan(
  examDate: Date,
  domains: KnowledgeDomain[],
  config: ExamConfig
): PlanTask[] {
  const totalDays = daysBetween(new Date(), examDate)
  const phaseDays = {
    foundation: Math.floor(totalDays * 0.60),
    reinforcement: Math.floor(totalDays * 0.30),
    sprint: totalDays - Math.floor(totalDays * 0.60) - Math.floor(totalDays * 0.30)
  }
  const tasks: PlanTask[] = []
  let currentDay = new Date()

  // 阶段 1: 基础 — 逐章学习
  const sortedDomains = topologicalSortByLevel(domains, 3)
  for (let d = 0; d < phaseDays.foundation; d++) {
    // 按大纲顺序，每次分配一个章节的阅读+练习
    const domain = sortedDomains[d % sortedDomains.length]
    const readingTask = createTask({ type: 'reading', domain, date: addDays(currentDay, d),
      estimatedMin: domain.suggested_min * 0.6 })
    const practiceTask = createTask({ type: 'practice', domain, date: addDays(currentDay, d),
      estimatedMin: domain.suggested_min * 0.4, questionCount: 15 })
    tasks.push(readingTask, practiceTask)
    // 检查每日时长上限
    if (dailyTotal(tasks, addDays(currentDay, d)) > config.daily_max_minutes) {
      // 顺延至下日
      moveExcessToNextDay(tasks, d)
    }
    // 每 7 天预留 1 天弹性日
    if ((d + 1) % 7 === 0) d++ // skip buffer day
  }

  // 阶段 2: 强化 — 弱项突破 + 案例/论文专项
  currentDay = addDays(currentDay, phaseDays.foundation)
  const weakDomains = getWeakDomains(domains) // 从练习记录中取正确率最低 30%
  for (let d = 0; d < phaseDays.reinforcement; d++) {
    const domain = weakDomains[d % weakDomains.length]
    tasks.push(createTask({ type: 'practice', domain, date: addDays(currentDay, d),
      difficulty: 'medium', questionCount: 10, includeMultipleChoice: true }))
    if (d % 3 === 0) {
      tasks.push(createTask({ type: 'review', domain: '案例分析专项', date: addDays(currentDay, d) }))
    }
    if (d % 7 === 0) {
      tasks.push(createTask({ type: 'essay', date: addDays(currentDay, d),
        topic: getNextHighFreqTopic() }))
    }
  }

  // 阶段 3: 冲刺 — 全真模考 + 复盘
  currentDay = addDays(currentDay, phaseDays.reinforcement)
  for (let d = 0; d < phaseDays.sprint; d += 2) {
    tasks.push(createTask({ type: 'mock_exam', date: addDays(currentDay, d),
      estimatedMin: 180, timed: true }))
    tasks.push(createTask({ type: 'review', date: addDays(currentDay, d + 1),
      subType: 'mock_review' }))
  }

  return tasks
}
```

#### 11.2 动态调整：速度感知

```typescript
function checkPaceAndAdjust(
  recentLogs: LearningLog[],
  planTasks: PlanTask[],
  config: ExamConfig
): AdaptAdjustment[] {
  const last3Days = groupByDate(recentLogs.slice(-3))
  const adjustments: AdaptAdjustment[] = []

  for (const [date, logs] of last3Days) {
    const actualMin = sum(logs.map(l => l.focus_minutes))
    const plannedMin = sum(planTasks.filter(t => t.date === date).map(t => t.estimated_min))

    if (actualMin < plannedMin * 0.7) {
      // 连续滞后：压缩后续低权重章节任务
      const futureTasks = planTasks.filter(t => t.date > date && t.priority < 2)
      for (const task of futureTasks) {
        if (task.task_type === 'practice') {
          task.estimated_min = Math.max(15, task.estimated_min * 0.7)
          task.linked_question_ids = task.linked_question_ids.slice(0, 8)  // 减量
        }
      }
      adjustments.push({ type: 'pace_warning', date, actualMin, plannedMin,
        action: 'compressed_low_priority' })
    }

    if (actualMin > plannedMin * 1.2) {
      // 超前：建议提前进入下一阶段或拓展阅读
      adjustments.push({ type: 'ahead_schedule', date,
        suggestion: '可提前进入下一阶段或生成拓展阅读任务' })
    }
  }

  return adjustments
}
```

```typescript
function generateDailyTop10Card(
  questionStats: TagStats[],
  recentErrors: AnswerRecord[],
  highFreqTags: Map<string, number>
): SprintCard {
  // 综合三个维度得分排序
  const scores = new Map<string, number>()
  for (const tag of allKnowledgeTags) {
    const errorRate = getErrorRate(recentErrors, tag)           // 用户错误率（权重 0.4）
    const examFreq = normalizeFrequency(highFreqTags.get(tag))   // 考试频率（权重 0.35）
    const lowMastery = 1 - (getMastery(questionStats, tag))     // 掌握度倒数（权重 0.25）
    scores.set(tag, errorRate * 0.4 + examFreq * 0.35 + lowMastery * 0.25)
  }
  const top10 = [...scores.entries()]
    .sort((a, b) => b[1] - a[1])
    .slice(0, 10)
    .map(([tag]) => ({
      tag,
      keyPoints: getKeyPointsFromOutline(tag),
      relatedQuestions: getRecentErrorsByTag(tag, 3),
    }))
  return { date: todayStr(), items: top10, generatedAt: new Date().toISOString() }
}
```

```typescript
function checkNotificationTriggers(db: Database): Notification[] {
  const newNotifications: Notification[] = []

  // 1. 每日计划通知（在学习开始时间前 30 分钟触发）
  const config = getExamConfig(db)
  const now = new Date()
  const startTime = parseTime(config.study_start_time)
  if (isWithinMinutes(startTime, now, 30)) {
    const todayTasks = getTodayTasks(db, getActivePlan(db)!.id)
    if (todayTasks.length > 0) {
      newNotifications.push({
        type: 'daily_plan',
        title: '今日学习计划',
        body: `共 ${todayTasks.length} 项任务，预计 ${sumEstimate(todayTasks)} 分钟`,
      })
    }
  }

  // 2. 进度滞后警告（连续 2 天完成 < 70%）
  const last2DaysCompletion = getLastNDaysCompletionRate(db, 2)
  if (last2DaysCompletion.every(r => r < 0.7)) {
    newNotifications.push({
      type: 'progress_warning',
      title: '进度滞后提醒',
      body: '连续 2 天未完成 70% 计划任务，建议调整计划或利用弹性日补漏',
      action_url: '/plans',
    })
  }

  // 3. 连续打卡里程碑
  const streak = calculateStreak(db)
  if ([3, 7, 21, 30].includes(streak)) {
    newNotifications.push({
      type: 'streak_milestone',
      title: `🎉 连续学习 ${streak} 天！`,
      body: getStreakMessage(streak),
    })
  }

  // 4. 考前倒计时
  const daysLeft = daysUntilExam(config.exam_date)
  if ([30, 14, 7, 3, 1].includes(daysLeft)) {
    newNotifications.push({
      type: 'countdown',
      title: `距考试还有 ${daysLeft} 天`,
      body: daysLeft <= 7 ? '最后冲刺阶段，保持节奏，注意休息！' : '合理安排复习计划，稳扎稳打',
    })
  }

  // 写入数据库并返回
  for (const n of newNotifications) {
    insertNotification(db, n)
  }
  return newNotifications
}
```

| 通知 | Electron `Notification` API (主进程) | 系统原生通知，支持点击跳转 |
| AI 学习顾问 Prompt | 结构化 system prompt + 用户数据 JSON 注入 | 确保 AI 基于实际数据回答，避免幻觉 |

---

### 14. 风险与注意事项

1. **大纲数据准确性**：`outline-sys-architect.json` 需对照官方最新版大纲人工校验。建议预留 1 天数据审核时间。

2. **计划生成与用户实际进度的冲突**：自动调整逻辑应在每次调整前保存"快照"，允许用户回退到上一版本计划。

3. **通知频率控制**：避免通知轰炸，同一类型通知每日最多触发 1 次，聚合多个提醒为一条每日摘要。

4. **大型计划性能**：当 `plan_tasks` 表超过 500 条时，前端日历视图需做虚拟滚动或按月分页加载。现有 `getCalendar()` 已按月查询，但 `getPlanTasks()` 需加 `LIMIT` + 分页。

5. **AI 建议的准确性**：学习顾问回答须在 prompt 中强调"基于提供的用户数据"，并附带免责声明"AI 建议仅供参考"。

6. **向后兼容**：所有 migration 仅 `ALTER TABLE ADD COLUMN`（带默认值）或 `CREATE TABLE IF NOT EXISTS`，不修改现有列或删除数据。`generatePlanTasks()` 保留旧签名作为 fallback。

7. **番茄钟中断检测的电池影响**：窗口焦点轮询间隔建议 5 秒而非实时，降低 CPU 占用。
