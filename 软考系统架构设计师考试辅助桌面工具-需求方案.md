# 软考系统架构设计师考试辅助桌面工具——需求方案（单机版）

## 1. 项目背景与目标
软考“系统架构设计师”属于高级资格考试，涉及知识面广、难度大，考生普遍面临备考资料分散、缺乏主观题批改反馈、学习计划难以坚持等痛点。本工具旨在打造一款**单机运行的 Windows 桌面应用**，将所有学习数据完全存储在用户本地计算机，无需任何云端账户系统，为用户提供从知识导入、智能出题、自动评分到学习规划与督促的一站式备考环境，提升学习效率与通过率。

## 2. 功能性需求

### 2.1 文档导入与知识解析
- **PDF导入**：支持导入官方教材、辅导书、讲义等 PDF 文件。对文本型 PDF 直接提取文字；对扫描型 PDF 可调用本地 OCR 引擎进行识别（如 Tesseract 或 Windows 内置 OCR）。
- **智能分段与归类**：利用 AI（本地模型或用户配置的远程 API）对提取文本进行章节识别、知识点划分，自动关联到考试大纲的相应考点标签（如“架构设计方法”、“质量属性”、“分布式系统”等）。
- **文档管理**：提供文件库视图，支持按科目、来源、导入时间筛选，支持全文检索，方便随时查阅原始资料。
- **知识图谱生成（进阶）**：基于导入文档自动构建知识点之间的关联关系，例如“微服务架构”与“服务治理”、“容器编排”的关联，辅助结构化学习。

### 2.2 AI 智能出题与评分
- **模拟题生成**：
  - 根据用户选择的知识点范围、题型（单选、多选、案例分析、论文）、难度等级，由 AI 基于已导入教材和本地题库生成新题。
  - 案例分析题和论文题可给出多个不同场景的命题，并提供参考答案要点或评分标准。
  - 支持一键生成整卷（综合知识 + 案例分析 + 论文），模拟真实考试结构。
  - AI 生成题支持保存到现有题库分组，或自动新建 AI 题目分组，便于后续按“AI 出题批次”筛选与练习。
- **主观题 AI 评分**：
  - 用户提交案例分析答案或论文后，AI 从“内容完整性、逻辑清晰度、专业术语运用、符合评分标准”等维度给出评分、评语和改进建议。
  - 支持人工复核模式，用户可查看 AI 评分依据，并可对争议评分申请二次分析。
- **客观题自动判定**：单选、多选即时出分，并显示知识点解析和出处（链接到导入的教材章节）。

### 2.3 网络题目提取
- **数据源管理**：预设一批可公开访问的软考相关题库网站、博客、论坛列表，用户可自定义添加或禁用。
- **智能爬取与清洗**：工具根据用户配置的规则，定时或手动执行爬取任务，抓取题目、选项、答案及解析。内置清洗引擎过滤广告、排版错误，去重后存入本地题库。
- **分组归档**：爬虫写入题库前，用户可选择保存到现有分组或新建分组；若抓取的是历年真题，分组需支持填写年份和上/下半年。
- **合规与频率控制**：严格遵守 robots.txt 协议，支持用户自定义访问间隔；工具仅提供基础 HTTP 请求与解析能力，不内置任何绕过反爬机制的手段。所有提取内容仅用于个人学习，不涉及二次分发。
- **手动剪藏**：提供浏览器扩展或系统级划词采集工具，用户在网页上浏览题目时可一键保存至本地题库。

### 2.4 学习计划与督促提醒
- **考试倒计时与大纲规划**：用户设置考试日期，系统根据官方大纲自动拆解各章节建议学习时长，生成每日/每周学习任务（如“周三前掌握架构评估方法，完成 20 道相关练习”）。
- **自定义计划**：允许手动调整章节顺序、增减任务、设定休息日。支持导入通用备考计划模板。
- **进度追踪**：以日历、进度条、统计图表展示整体完成度、每日投入时间、做题正确率变化趋势。
- **智能提醒**：
  - 桌面通知提醒每日学习任务、未完成计划。
  - 考前冲刺阶段自动提高提醒频率，推送薄弱知识点复习建议。
  - 支持番茄钟模式，学习时段内免打扰并记录专注时长。
- **激励与成就系统**：完成阶段目标、连续打卡、正确率提升等给予成就徽章，增强学习动力。

### 2.5 练习与模考
- **题库分组与筛选**：题库需支持分组管理，至少包括“历年真题”“AI 出题”“爬虫导入”“自定义分组”等类型；历年真题分组需支持年份和上/下半年标记，题库练习和查询时可按这些条件筛选。
- **题库练习**：顺序练习、随机练习、错题重做、收藏题目专项练等模式，并支持按分组、年份、上/下半年、来源筛题。
- **全真模拟**：按照考试时间、题型比例、分值分布组卷，全程计时，交卷后客观题立即出分，主观题流转至 AI 评分。
- **历史记录与错题本**：自动记录每道题的作答情况，错题本按知识点分类，支持一键重做或导出。
- **成绩分析报告**：从知识域、题型、难度等多维度分析强弱项，生成雷达图、柱状图，指导后续复习重点。

### 2.6 AI 对话与答疑（增值功能）
- 基于已导入的教材内容和题库，提供问答界面，用户可对某个概念、题目直接提问，AI 给出解答并标注来源页码或链接。
- 支持追问和澄清，模拟一对一辅导。

## 3. 非功能性需求

### 3.1 用户界面与交互
- 原生 Windows 桌面应用体验，采用 WPF 或 Electron 框架，遵循 Windows 设计规范，支持高 DPI、暗黑模式。
- 主界面采用多面板布局：导航栏（题库、学习计划、文档、成绩、AI 助手）、内容区、状态栏。学习提醒以系统通知和界面悬浮卡片形式出现。
- 所有文件操作应有明确的进度条与状态反馈，大数据量题库加载支持分页与虚拟滚动。

### 3.2 数据安全与隐私
- 所有用户文档、题库、学习记录**默认且仅存储在本地加密数据库**（如 SQLite with encryption）中，不上传至任何云端服务器。
- AI 功能若涉及调用远程大模型 API，用户需自行配置 API 密钥并在本地发起请求，工具本身不收集或中转任何学习数据；同时支持接入本地部署的大模型（如通过 Ollama），实现纯离线的 AI 能力。
- 网络爬取功能需提示用户注意目标网站的版权和服务条款，由用户自行判断合规性。

### 3.3 性能与可靠性
- 文档解析与 AI 出题等耗时任务放入后台线程或异步处理，不阻塞 UI。
- 题库容量设计应支持至少 5 万道题目而不出现明显卡顿，搜索响应时间 <500ms。
- AI 服务异常或网络中断时，应有友好的降级提示，本地所有非 AI 功能仍可正常使用；若配置了本地模型，则离线 AI 功能可继续运行。

### 3.4 可扩展性与维护性
- AI 能力通过可插拔接口对接不同大模型（OpenAI、百度文心、阿里通义、本地 Ollama 等），用户可在设置中切换并填入自己的 API 密钥。
- 题库数据格式标准化，支持导出为 JSON、CSV 等通用格式，便于迁移与备份；导出时需保留题目所属分组、真题年份和上/下半年等元数据。
- 学习计划模板、爬虫规则、评分维度等采用配置文件化，方便用户自定义与分享（不含版权内容）。

## 4. 技术实现约束（选型建议，非具体实现）
- **运行环境**：Windows 10/11 x64，需 .NET Framework 4.8+ 或 Electron 对应 Node.js 运行时。内存建议 8GB 以上。
- **AI 接入**：优先支持 OpenAI 兼容接口，并预置国内合规大模型配置向导。本地推理可选用 Ollama + 7B/13B 规模的量化模型。
- **PDF 处理**：后端可集成 PdfPig、iTextSharp 或调用 Python 服务；OCR 可选用 Windows.Media.Ocr 或 Tesseract 引擎。
- **数据存储**：SQLite + 全文检索扩展（FTS5）用于文档和题库索引。
- **网络爬虫**：内嵌轻量级 HTTP 客户端，配合 HtmlAgilityPack 或 AngleSharp 解析页面，支持简单正则和 CSS 选择器规则。

## 5. 针对功能的需求建议

### 5.1 提升 AI 拟真度与专业性
- 针对系统架构设计师的论文评分，应内置历年评分标准细项（如摘要完整性、正文深度、项目实践结合度），让 AI 依据明确规则评分，而非简单流畅度判断。
- 生成模拟题时，支持“基于高频考点”、“结合当前技术趋势（如云原生、数字孪生）”等命题倾向，可由用户勾选。

### 5.2 学习计划的自适应调整
- 系统应能根据用户做题正确率和学习速度，动态调整后续计划。例如，质量属性章节正确率持续低于 60%，则自动增加该章节的练习量和复习频次，并推迟其他非重点任务。
- 支持“冲刺模式”：距考试 30 天内，自动切换为以错题、案例分析和论文训练为主，减少知识记忆型题目比重。

### 5.3 爬虫规则的自定义与导入导出
- 考虑到不同网站频繁改版，用户可自行编写、调整爬取规则（基于正则或 CSS 选择器），并支持将规则导出为文件进行备份或与他人分享（仅分享规则，不含题库内容）。提供“规则测试”功能，帮助验证修改后的有效性。
- 浏览器扩展“一键抓题”：通过 DOM 选择器由用户手动指定题目区域，系统自动学习模式，生成抓取模板。

### 5.4 强化论文专项训练
- 论文作为高权重科目，应获得专项支持。可设计“论文结构引导式编辑器”，提供摘要、背景、论点、实践、总结等分段模板，AI 实时对每部分给出字数统计、术语建议和方向提示。
- 建立“论文素材库”，从导入的项目文档或用户自行记录的工作经验中提取可用案例要素，方便在写作时引用。

### 5.5 数据备份与恢复
- 支持手动或定时自动备份学习数据（题库、进度、配置）至本地指定目录、移动存储或用户配置的 WebDAV 云存储。备份文件为加密压缩包，确保隐私。
- 提供“从备份恢复”功能，可将任意备份文件恢复到当前环境，方便更换电脑后继续学习，但不支持多设备实时同步。

### 5.6 健康学习提醒
- 持续学习超过 45 分钟时，弹出眼保健操或起身活动提醒，避免疲劳。
- 显示每日学习时长统计，建议合理分配，不超过系统推荐的上限，防范过度学习。

### 5.7 合规性与伦理
- 明确提示用户：AI 评分和生成内容仅供参考，不作为最终考试评判依据；用户有义务使用正版教材，工具仅提供辅助，不鼓励盗版传播。
- 网络爬取功能需经用户确认合规性，工具不内置绕过反爬机制的手段，仅提供基础的 HTTP 请求与解析能力。

## 6. 验收标准（概要）
1. 能够成功导入典型官方教材 PDF（大于 200 页），并正确提取文本，知识点归类准确率 >80%。
2. AI 能够生成与考试题型、难度匹配的模拟题，主观题评分与人工评分的偏差在可接受范围（如论文分数偏差 ≤5 分/总分 50）。
3. 从指定公开题库网站稳定抓取题目，去重准确，无重复条目。
4. 学习计划自动生成符合大纲要求，每日提醒按时触发，学习数据统计准确。
5. 在标准配置 PC 上，连续运行 2 小时无内存泄漏，UI 操作流畅。
6. 断网条件下，本地题库练习、文档查看、已生成试题练习等离线功能完全可用；如配置了本地 AI 模型，智能出题与评分功能亦可离线使用。
